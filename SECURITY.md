# Security Policy

## Scope

Este repositorio contiene documentación, plantillas y scripts para gestionar skills y pipelines de contenido.

## Reporting a Vulnerability

Si encuentras un problema de seguridad:

1. No lo publiques en un issue abierto.
2. Repórtalo de forma privada al mantenedor del repositorio.
3. Incluye pasos de reproducción, impacto y alcance conocido.

## What Counts as a Security Issue Here

Ejemplos relevantes:

- exposición accidental de secretos o credenciales,
- ejecución insegura en scripts del repositorio,
- workflows de GitHub Actions con riesgos claros de supply chain,
- archivos o procesos que puedan filtrar información sensible.

## Current Limitations

Este repositorio todavía no define un canal público formal de disclosure ni SLA de respuesta. Antes de abrirlo ampliamente, conviene añadir esa información.
