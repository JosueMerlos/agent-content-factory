# Tests

## Test 1: No asume licencias inexistentes

La prueba falla si el análisis trata material no documentado como seguro.

## Test 2: La decisión final es accionable

Debe terminar en `publish`, `revise` o `block`, con razón y siguientes pasos.

## Test 3: Los riesgos están aterrizados

Cada riesgo debe estar vinculado a un asset o categoría concreta.

## Test 4: Hay mitigaciones reales

La salida debe proponer reemplazar, recortar, relicenciar o documentar permiso cuando corresponda.
