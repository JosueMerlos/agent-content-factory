# Text to Video for OpenCode

Usa esta skill para crear un plan visual que el usuario pueda entender directamente y usar para editar.

## Instrucciones

1. Describe las escenas en lenguaje natural, no en tablas técnicas.
2. El usuario debe poder leer esto y saber qué poner en cada momento del video.
3. Incluye lo necesario para producir: footage, texto en pantalla, transiciones, música.

## Formato de salida

### El estilo visual
Describe cómo se ve el video:
- Tono general
- Tipo de transiciones
- Uso de texto en pantalla

### Escenas en orden
Para cada momento clave:
- Qué pasa (descripción simple)
- Qué se ve (footage, gráficos, texto)
- Cuánto dura
- Cómo transiciona

### Lo que necesitas conseguir
- Footage a buscar
- Música o efectos
- Gráficos

### Notas de edición
Puntos importantes para editar bien.

## Regla clave

La salida debe ser **fácil de leer y usar**. Evita estructuras técnicas innecesarias.
Al final, si necesitas metadata técnica, inclúyela de forma separable.
