# GitHub Release Checklist

Usa esta lista antes de publicar el repositorio:

1. Confirmar que `LICENSE` refleje la licencia deseada del proyecto.
2. Revisar que `README.md` describa el estado real del proyecto.
3. Ejecutar `./scripts/validate-skills`.
4. Confirmar que no haya secretos, credenciales o contenido privado.
5. Verificar que ejemplos y tests manuales estén presentes en nuevas skills.
6. Confirmar que GitHub Actions pase correctamente.
7. Revisar si conviene agregar `CODEOWNERS`, `SECURITY.md` o `CODE_OF_CONDUCT.md`.
